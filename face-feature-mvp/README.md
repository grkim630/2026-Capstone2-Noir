# Face Feature MVP

`face-feature-mvp`는 NOIR 프로젝트의 얼굴 분석 핵심 로직을 담고 있는 Python 기반 MVP 모듈입니다. 웹 서버의 분석 API는 이 폴더의 모듈을 사용해 촬영 이미지에서 얼굴 랜드마크, 피부 색, 입술/볼/이마/눈 주변 색, 얼굴 비율 특징을 추출합니다.

## 주요 역할

- MediaPipe Face Mesh로 얼굴 랜드마크를 감지합니다.
- 얼굴 부위별 마스크를 만들고 피부/입술/볼/이마/눈 주변 색을 추출합니다.
- RGB 값을 LAB/HSV로 변환해 색온도, 명도, 채도 분석에 사용할 수 있게 만듭니다.
- 촬영 프레임이 분석 가능한 품질인지 확인합니다.
- 여러 프레임의 결과를 종합해 더 안정적인 대표 분석값을 만듭니다.
- 메이크업 여부에 따라 피부 분석 가중치와 결과 해석을 조정합니다.

## 기술 스택

- Python
- OpenCV
- MediaPipe

## 주요 파일 설명

### `main.py`

웹캠 또는 이미지 프레임 기반 분석 흐름을 실행하는 메인 파일입니다.

처리 흐름:

1. 프레임을 입력받습니다.
2. 얼굴 랜드마크를 감지합니다.
3. 얼굴 정렬, 밝기, 검출 품질 등 분석 가능 여부를 확인합니다.
4. 부위별 색상과 얼굴 특징을 추출합니다.
5. 여러 프레임이 있으면 안정적인 값으로 집계합니다.
6. JSON 결과를 저장합니다.

### `run_from_file.py`

이미지 파일 하나를 입력해 분석 결과를 확인할 때 사용하는 실행 파일입니다. 서버 연동 없이 로컬에서 분석 로직을 테스트하기 좋습니다.

### `modules/landmark.py`

MediaPipe Face Mesh를 사용해 얼굴 랜드마크를 감지합니다. 이후 모든 부위 분석은 이 랜드마크 좌표를 기준으로 진행됩니다.

### `modules/color_extractor.py`

얼굴의 주요 부위 색상을 추출합니다.

추출 대상:

- 입술 색
- 볼 색
- 이마 색
- 눈 주변 색
- 홍채/눈 관련 색상
- 피부 베이스 색상

이마 분석은 중앙 이마를 우선 확인합니다. 중앙 이마가 머리카락, 그림자, 가림 등으로 이상하다고 판단되면 확장 이마 영역을 확인하고, 그래도 신뢰하기 어렵다면 전체 피부 베이스톤을 대신 반영합니다. 이 방식은 이마 전체가 너무 넓어 분석값이 흔들리는 문제를 줄이기 위한 장치입니다.

### `modules/skin_mask.py`

피부색 분석에 사용할 얼굴 영역 마스크를 만듭니다. 눈, 눈썹, 입술처럼 피부색 판단을 방해할 수 있는 영역은 제외하고, 볼/이마/턱 등 피부 영역을 중심으로 픽셀을 모읍니다.

### `modules/skin_cluster.py`

피부 픽셀에서 대표 색상을 찾기 위해 클러스터링을 수행합니다. 단순 평균보다 조명이나 잡색에 덜 흔들리는 대표 피부색을 얻기 위한 단계입니다.

### `modules/skin_analysis.py`

NOIR 프로젝트의 피부 톤 분석 핵심 파일입니다. RGB 기반 대표 피부색을 LAB/HSV 색공간으로 변환한 뒤 다음 요소를 분석합니다.

- 색온도: 웜, 뉴트럴, 쿨
- 명도: 밝음, 중간, 딥
- 채도: 낮음, 중간, 높음
- 홍조/붉은기
- 멜라닌/어두운 정도
- 전체 퍼스널 컬러 방향성

## 얼굴 분석 방식

### 1. 얼굴 감지와 정렬 확인

MediaPipe Face Mesh가 얼굴 랜드마크를 찾으면, 얼굴이 화면 안에 충분히 들어왔는지와 정면에 가까운지를 확인합니다. 너무 기울어졌거나 얼굴이 잘린 프레임은 분석에서 제외하거나 낮은 신뢰도로 처리합니다.

### 2. 부위별 색상 추출

얼굴 전체를 하나의 평균색으로 보지 않고, 부위별 의미를 나누어 분석합니다.

- 볼: 실제 피부 베이스와 혈색을 함께 반영합니다.
- 이마: 머리카락 영향을 받기 쉬워 중앙 이마를 우선 사용합니다.
- 턱/피부 베이스: 화장품이 덜 올라간 안정적인 피부색 후보로 사용합니다.
- 입술: 립 컬러 또는 본래 입술색을 파악합니다.
- 눈 주변: 아이 메이크업 영향과 얼굴 인상 분석에 참고합니다.

### 3. 메이크업 여부 반영

PC Client에서 사용자가 메이크업 여부를 선택하면 서버가 이 값을 분석 모듈에 전달합니다.

메이크업이 있다고 판단되는 경우:

- 파운데이션이 피부 베이스를 바꿀 수 있으므로 분석 결과는 "현재 보이는 베이스톤"으로 해석합니다.
- 립, 블러셔, 아이 메이크업처럼 국소적으로 강한 색은 피부 베이스 판단에서 과도하게 반영되지 않도록 가중치를 조정합니다.
- 특정 부위가 메이크업 영향을 받은 것으로 보이면 metadata에 표시합니다.

이 프로젝트는 촬영된 얼굴에서 "완전한 민낯 피부"를 복원하려고 하기보다, 현재 카메라에 보이는 얼굴 상태를 기준으로 추천 경험을 구성합니다.

## 색상 판정 기준

분석에는 LAB와 HSV 색공간이 함께 사용됩니다.

### 색온도

LAB의 `a`, `b` 축 차이를 사용합니다.

- `b - a`가 충분히 크면 웜 방향
- `a - b`가 충분히 크면 쿨 방향
- 둘의 차이가 작으면 뉴트럴 방향

### 명도

LAB의 `L` 값을 사용합니다.

- 높은 L 값: 밝은 피부 톤
- 중간 L 값: 중간 명도
- 낮은 L 값: 딥한 명도

### 채도

HSV의 `S` 값을 사용합니다.

- 높은 S 값: 선명한 색감
- 낮은 S 값: 부드럽거나 차분한 색감

## 추천 시스템과의 연결

이 폴더는 제품 추천을 직접 결정하지는 않습니다. 대신 서버가 추천을 계산할 수 있도록 다음과 같은 분석 결과를 제공합니다.

- 피부 대표 RGB/HEX
- LAB/HSV 값
- 색온도/명도/채도 분류
- 부위별 색상
- 메이크업 영향 여부
- 분석 품질 metadata

서버의 `recommendation` 모듈은 이 결과와 `data/noir_color_db.json`의 제품 색상 DB를 비교해 어울리는 립, 블러셔, 아이 팔레트를 계산합니다.

## 설치 및 실행

```bash
cd face-feature-mvp
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

웹캠 기반 테스트:

```bash
python main.py
```

이미지 파일 기반 테스트는 `run_from_file.py`를 사용합니다. 세부 인자는 파일 내부의 실행 옵션을 확인해 사용합니다.

## 출력 데이터

분석 결과는 `outputs/` 폴더에 JSON 형태로 저장됩니다. 이 폴더는 테스트 결과가 계속 쌓이는 위치이므로 GitHub에는 올리지 않는 것을 권장합니다.
